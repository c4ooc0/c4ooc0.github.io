VScode 配置远程开发C语言程序环境

发表于 2021-09-28 更新于 2022-11-17 本文字数： 419 阅读时长 ≈ 2 分钟

有些坑不记得了，如果哪里不完善，可以联系我 （从笔记里搬到博客凑数

  1. 插件
  • C/C++
  • code Runner
  • Run in Terminal
  • Remote - SSH
  1. 配置 ssh 服务
    1. 服务端配置 ssh 服务（以 centos8 为例，需要确保两台主机能互相 ping 通）
  1. vscode ssh 登录服务端

安装完 Remote - SSH 会有这个图标，点击->Connect to Host->Configure SSH Hosts->选择一个 config

  • 配置 config 的信息：

Host：配置文件的名字 HostName：远程主机的 IP 地址 User：登录用户

  • 右击->Connect to Host

  • 选择对应的平台

  • 输入密码，按回车
  1. 配置 C 语言环境
  2. 从菜单栏选择 Terminal>Configure Default Build Task, 在下拉栏里选择 C/C++: g++ build active file. 这会生成 tasks.json 文件
  3. 在菜单栏选择 Debug>Add Configuration, 选择 C++ (GDB/LLDB), 在下拉栏中选择 g++ build and debug active file.这会创建 launch.json
  4. 在 setting.json 里添加"editor.formatOnType": true,（c、d 两步是为了自动缩进和方便输入）
  5. 更改设置 文件->首选项->设置

搜索 Run In Terminal ，勾选 搜索 format，勾选

  1. 之后应该就能跑了，最近事务繁多，无暇再配置一遍看看有没有坑了，总之有坑联系我

第0期 DVWA安装部署（Windows）

发表于 2021-06-20 更新于 2022-11-17 本文字数： 414 阅读时长 ≈ 2 分钟

Damn Vulnerable Web Application（DVWA）是一个非常脆弱的 PHP/MySQL Web 应用程序。它的主要目标是帮助安全专业人员在合法环境中测试他们的技能和工具，帮助 web 开发人员更好地理解保护 web 应用程序的过程，并帮助学生和教师在受控教室环境中学习 web 应用程序安全。 看 DVWA 首页似乎推荐部署在虚拟机里，有机会改一下

第一步 下载 phpstudy 并安装

https://www.xp.cn/download.html 解压后进行安装，一般装在 D 盘问题不大。 安装完成后，浏览器访问127.0.0.1，看到如下界面即安装成功。

第二步 下载 DVWA 源码

https://github.com/digininja/DVWA 或者使用命令： git clone https://github.com/digininja/DVWA.git 打开 phpstudy 网站的根目录，在 phpstudy 的安装目录下，路径一般是...\phpstudy_pro\WWW。 将解压后的 DVWA 文件夹复制到该目录。方便期间，DVWA-master 文件可以改名为 DVWA，以下以改名为前提进行演示。 DVWA->config->config.inc.php.dist 文件，修改后缀名为 ini.php。  **-> ** 打开该文件，修改数据库密码。 首先查看 phpstudy 中的数据库密码，鼠标移到密码处就显示了。 将 db_user 改为 root，db_password 根据 phpstudy 显示的改，我这里是 123456。

第三步 创建数据库

浏览器访问127.0.0.1/DVWA/setup.php，找到最后，创建数据库。(这里如果失败，一般就是数据库名或者密码不对，看一下 phpstudy 的数据库设置就知道了)   ->  访问http://127.0.0.1/DVWA/index.php 进行登录，用户名/密码：admin/password。登陆成功后，DVWA 就可以正常使用了。
